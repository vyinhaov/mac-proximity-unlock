# PRD-001: Mac 近距离感应自动解锁

> 状态: 待架构师评审 | 优先级: P0 | 项目: mac-proximity-unlock

## 1. 项目目标

两台 MacBook（本机 + 远程 `x.x.x.x`），同 Apple ID，同局域网段。通过距离感应实现：
- **靠近解锁**：本机跟随远程 MacBook 的解锁状态自动解锁
- **远离锁屏**：距离拉开后本机自动锁屏
- **再次靠近解锁**：靠近且远程已解锁时，本机自动解锁

## 2. 用户故事

| 场景 | 触发条件 | 本机动作 |
|-----|---------|---------|
| 带着 MacBook 回到工位 | 距离 近 + 远程 解锁 | 本机自动解锁 |
| 远程在面前，解锁远程 | 距离 近 + 远程 解锁状态变化 | 本机跟随解锁 |
| 带着 MacBook 离开 | 距离 近→远 | 本机自动锁屏（若未锁） |
| 远程保持锁定 | 距离 近 + 远程 锁定 | 本机不动作 |
| 远程锁屏 | 距离 近 + 远程 锁定状态变化 | 本机跟随锁屏 |

## 3. 功能需求

### 3.1 距离感知

**双信号融合**，解决蓝牙单独使用不稳定的问题：

| 层 | 技术 | 作用 | 精度 |
|---|-----|------|------|
| 主要 | 蓝牙 RSSI | 近距离精度高（0-10m） | 通过 Kalman 滤波平滑 |
| 辅助 | ICMP Ping 延迟 | 同局域网 0.3-1ms = 很近, >5ms = 较远 | 粗略，但稳定 |

**融合策略**：
- 蓝牙 RSSI > -55dBm **且** ping < 2ms → 判定为"近"
- 仅一方满足 → 维持当前状态，避免误触发
- 连续 3 次判定为"远" → 触发远离锁屏

### 3.2 远程状态监控

通过 SSH 免密连接远程 MacBook，周期性查询锁屏状态：

```shell
# macOS 屏幕锁定检测（在远程执行）
python3 -c "import Quartz; d = Quartz.CGSessionCopyCurrentDictionary(); print('locked' if d.get('CGSSessionScreenIsLocked', 1) else 'unlocked')"
```

同时监控 `CGSSessionScreenIsLocked` 的变化事件（通过 `log stream` 监听），实现实时响应而非轮询。

### 3.3 本地动作执行

**锁屏**：
- 标准方式：`osascript -e 'tell application "loginwindow" to «event aevtlgo»'`
- 回退方式：模拟 `Ctrl+Cmd+Q`

**解锁**：
- 从 Keychain 读取用户密码
- 模拟 HID 键盘输入密码 + Enter
- 需要 Accessibility 权限（首次启动引导用户授权）

### 3.4 安全约束

- 密码仅存在本机 Keychain（`security add-generic-password -s "MacAutoUnlock" -a "$USER" -w "$PASSWORD"`）
- 仅当远程 MacBook 在近距离范围内且已解锁时，才触发本机解锁
- 网络通信仅检测屏幕锁定状态，不传输密码等敏感数据
- 离开后锁屏保证物理安全

## 4. 非功能需求

| 维度 | 要求 |
|-----|------|
| 响应速度 | 靠近 3 秒内解锁，远离 5 秒内锁屏 |
| CPU 占用 | 空闲 < 1%，RSSI 扫描轮询间隔 1s |
| 不稳定性 | 蓝牙抖动通过 Kalman 滤波消除 |
| 自启动 | LaunchAgent 开机自启 |
| 可观测性 | 日志写入 `~/Library/Logs/mac-auto-unlock/` |

## 7. 验收标准

1. 建立 SSH 免密连接后，能准确读取远程锁屏状态
2. 蓝牙 RSSI + Ping 融合判定距离，"近/远"区分准确
3. 靠近且远程解锁时，本机 3 秒内自动解锁
4. 远离后本机 5 秒内自动锁屏
5. 远程锁屏时本机不会误解锁
6. 密码存储在 Keychain 中，不硬编码
7. LaunchAgent 自启，重启后无需手动干预
